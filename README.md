# 💳 Credit Card Fraud Detection using Exploratory Data Analysis (EDA)

## 📌 Project Overview
This project performs Exploratory Data Analysis (EDA) and statistical analysis on a credit card transaction dataset to identify patterns associated with fraudulent transactions. The analysis focuses on understanding the characteristics of fraud, data distribution, correlations, and statistical significance to support fraud detection.

## 🎯 Objective
- Explore and understand credit card transaction data.
- Identify differences between fraudulent and legitimate transactions.
- Perform statistical analysis and hypothesis testing.
- Visualize transaction patterns using various plots.
- Generate insights that can help improve fraud detection systems.

## 📂 Dataset
The dataset contains anonymized credit card transactions with features generated using PCA. It includes transaction amount, time, and a target variable (`Class`) indicating fraud (1) or non-fraud (0).

> **Note:** The dataset is not included in this repository because it exceeds GitHub's web upload size limit. Place the `creditcard.csv` dataset in the project folder before running the notebook.

## 📊 Analysis Performed
- Data Cleaning

- Feature Scaling
- ANOVA

## 📈 Key Insights
- Fraudulent transactions represent only a very small percentage of the dataset.
- Transaction amount and several transformed features show different distributions for fraud and non-fraud cases.
- Correlation analysis helps identify relationships between important variables.
- Statistical testing supports identifying significant differences between fraudulent and normal transactions.
